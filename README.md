# My Human Space Activities Website

This project is a multi-page informational website focused on Human Space Activities. It provides an engaging overview of major historical milestones, current developments, and future missions in space exploration.

## Website Purpose

The purpose of this website is to:

- Present key historical space achievements  
- Highlight upcoming and future missions  
- Provide an accessible and intuitive browsing experience  
- Allow users to interact through a contact form  

The site is intended for students, educators, and anyone interested in space exploration.

---

## Technologies Used

- **HTML** – Semantic page structure  
- **CSS** – Styling, layout (Flexbox & Grid), and hover effects 
- **Google Maps Embed** – Location display on Contact page  

No external frameworks or libraries were used.

---

## Website Structure

The website consists of four main pages:

- `home.html` – Introduction and overview  
- `history.html` – Key historical milestones  
- `future.html` – Timeline of upcoming missions  
- `contact.html` – Contact form with validation and embedded map

## Design Decisions

- Consistent navigation bar and footer across all pages  
- Blue colour scheme for strong visual hierarchy  
- Arial font for clarity and readability  
- CSS hover effects for interactive feedback  
- Vertical milestone layout on History page  
- Central timeline layout on Future page  
- 4×3 image grid for visual content presentation  
- Embedded Google Map on Contact page  

---

## Form Validation

The Contact page includes:

- Required fields  
- HTML email validation  

This ensures accurate user input while keeping the implementation simple.

---

## Deployment

The website is deployed on a virtual machine (VM) environment. Files are transferred from a local machine and hosted using a web server.

Testing included:

- Verifying file paths  
- Checking asset loading (CSS, images, JS)  
- Ensuring navigation links function correctly  
- Validating form behaviour
